ETIQUETAS DE SECCIÓN (BLOQUES)

    body:
        Contenido de la página
    article:
        Contenido dentro de una sección. Debe tener
        un encabezado.
    h1-h6:
        Títulos
    section:
        Divide bloque de contenido
    aside:
        Contenido relacionado con el contenido
        principal
    header:
        Cabecera de un bloque de contenido
    footer:
        Pie de bloque de contenido
    nav:
        Enlaces de navegación de la página


IMPORTANTE: - tener etiqueta link con icon
            - No se puede utilizar un h2 sin 
            un h1
            - Es importante la semantica de las
            etiquetas.

Ejemplo:
    ARTICULO
        <body>
            <h1>H1</h1>
            <article>
                <h2>H2</h2>
                <section>
                    <h3>H3</h3>
                    <p>
                        texto
                    </p>
                </section>
            </article>
        </body>
    MENU
        <body>
            <header>
                <nav>
                    <span>menu 1</span>
                    <span>menu 2</span>
                </nav>
            </header>
        </body>
    
ETIQUETAS SEMANTICAS A NIVEL DE texto

    a: 
        link
    em:
        énfasis de su contenido (para navegador)
    strong:
        texto gran importancia (para el navegador)
    small:
        contenidos secundarios
    cite:
        referencia a un trabajo
    q:
        frase citada de otra fuente (1 linea)
    code:
        fragmento código
    sub:
        subíndice
    sup:
        superíndice
    span:
        no significa nada, se utiliza con CSS
    br:
        salto de línea (obligado)
    wbr:
        salto de linea opcional. (lo decide navegador)

Importancia de las etiquetas
    texto

        con strong: 2

IMÁGENES DINÁMICAS 
            
    cuando se quiere ajustar la imagen a un
    dispositivo especifico se utiliza el 
    atributo srcset="ruta"

    si el navegador no soporta srcset entonces tenemos
    src para ponerl a imagen
    <img src="ruta" alt="desc" srcset="rutas">

    rutas = grupo de imagenes separadas por ',' 

    NOTA:
        puedes indicarle el PIXEL RATIO o 

        el ancho de la imagen (1024w) -> esto 
        quiere decir que: SI EL VIEWPORT es 
        menor a 1024 utiliza esa imagen
    
    <img src="" alt="desc" srcset="
    img1.png 1x,
    img2.png 2x,
    img3.png 4x
    ">

    CONCEPTO IMPORTANTE

    DEVICE PIXEL RATIO
        relacion de aspecto entre los pixeles de la web y los reales
        PIXEL FISICO: son los que estan en la pantalla
        PIXEL LOGICO: no tiene dimensiones específicas
            sino que están establecidas por software

        En un dispositivo con PIXEL RATIO = 1 
        significa que el pixel logico ocupa lo mismo 
        que pixel fisico 

META VIEWPORT 

    <meta name="viewport" 
    content="width=device-width, initial-scale=1.0">

    width:
        estable ancho de la web, puede ser una medida 
        o device-width

    initial-scale: 
        establece escala inicial de la web
    
    maximun-scale:
        establece escala maxima
    minimun-scale:
        establece escala minima

     user-scalable: (yes o no) responsive
        establece si el usuario puede o no hacer zoom
